Ubuntu Command Simplification Tool(UCST)使用说明

简介
    Ubuntu Command Simplification Tool(UCST)是一个Bash脚本工具，旨在简化Ubuntu系统中的常用命令操作。通过封装复杂的Linux命令为简单易记的短命令（或以日常英语单词重新定义），适合Linux新手或希望提高效率的用户使用。
安装与设置
    工具在系统启动时会自动创建符号链接，无需手动安装。首次运行任何命令时会自动设置所有必要的链接。
    您可使用这个命令来验证符号链接是否正常创建：
        ls -la /usr/local/bin/command（在command处替换您需要验证的命令名称）
命令列表（在系统中可键入“helpUCST”来获取命令帮助）
    系统信息命令
        about        # 显示基本信息
        about -a     # 显示详细信息（硬件、网络、内存等）
        about -c     # 显示处理器信息
        about -g     # 显示显卡信息
        about -o     # 显示操作系统信息
        about -s     # 显示用户信息
        ctime        # 显示本地时间、时区、系统运行时间等信息
    文件管理命令
        list        # 显示当前目录文件列表
        list <路径> # 显示指定路径文件列表
        open <文件路径> # 打开指定文件查看内容
        new <路径> # 新建目录或文件
        delete <路径> # 删除指定的文件或目录
        mod <路径> # 修改文件或目录属性
    系统监控命令
        disk        # 查看磁盘使用情况
        disk -m     # 管理硬盘挂载（需要root权限）
        network -i  # 查看基本网络信息
        network -c  # 查看所有网卡及连接情况
        process list    # 查看进程列表（前10个）
        process all     # 查看所有进程
        nkill <进程ID>      # 强制杀死进程
        nkill -p <进程ID>   # 暂停进程
        nkill -c <进程ID>   # 继续被暂停的进程
        driver          # 列出当前已安装的驱动程序
        driver -r       # 扫描所有硬件驱动
        driver -i       # 自动安装缺失的驱动
        driver -i<名称> # 安装指定的驱动
        driver -d<名称> # 删除指定的驱动
        backup # 启动备份管理界面
        check         # 检查文件编码格式
        check -f      # 检查代码文件格式
        helpUCST      # 显示所有可用命令的简要说明
详细功能说明
    硬盘挂载管理 (disk -m)
        这是一个交互式工具，用于管理存储设备的挂载：
        1.扫描设备：自动扫描所有连接的存储设备
        2.设备选择：列出所有设备，包括已挂载和未挂载的
        3.操作选择：
              对于未挂载设备：询问挂载点，提供临时/永久挂载选项
              对于已挂载设备：提供卸载或重新挂载选项
        4.安全特性：
              需要 root 权限
              挂载点目录检查
              确认提示防止误操作
    系统信息查看 (about)
        提供全面的系统信息查看功能：
            1.基本信息：主机名、用户、发行版、内核、架构等
            2.硬件信息：处理器、显卡、内存、磁盘详情
            3.系统状态：运行时间、负载、网络配置
            4.环境检测：自动识别 WSL 或原生 Linux 环境
    备份系统 (backup)
        提供完整的备份解决方案：
            1.备份类型：完全备份、增量备份、差异备份
            2.备份历史：记录所有备份操作，支持按时间恢复
            3.恢复功能：从备份历史中选择恢复点
            4.日志记录：详细的备份操作日志
注意事项
    1.文件编码：若您重开发了该代码，需确保脚本文件使用Unix行尾符 (LF)
    2.重要操作（如删除、挂载）前建议备份数据
    3.该工具目前只通过了WSL环境下的Ubuntu24.04环境测试，其他Linux系统尚未能完成测试，请谨慎使用
故障排除
    常见问题
        1.命令未找到：运行 helpUCST 确认命令是否存在
        2.权限不足：按提示使用 sudo 前缀
        3.语法错误：检查文件编码和行尾符
